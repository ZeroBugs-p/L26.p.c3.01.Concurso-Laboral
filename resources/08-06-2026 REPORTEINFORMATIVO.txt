=== RESUMEN GENERAL DEL PROYECTO ===

El proyecto es una aplicación web tipo MVC con dos roles principales:
- Evaluador: ingresa información de aspirantes y califica sus puntajes.
- Administrador: visualiza las entregas registradas en una tabla.

El sistema usa:
- HTML para la interfaz.
- Vistas (`views`) para leer/escribir el DOM y exponer eventos.
- Controladores (`controllers`) para coordinar vistas, modelos y servicios.
- Modelos (`models`) para la lógica de negocio y cálculos.
- Servicios (`services`) para comunicarse con MockAPI.

También incluye documentación y comandos de apoyo en archivos de texto.

---

1) tsconfig.json

Este archivo configura el compilador TypeScript:
- `target: ES2022`: compila a código JavaScript moderno ES2022.
- `module: ES2022`: usa módulos ES nativos.
- `rootDir: ./src`: la raíz del código fuente está en `src`.
- `outDir: ./dist`: los archivos compilados se escriben en `dist`.
- `noEmitOnError: true`: no genera `dist` si hay errores de compilación.
- `esModuleInterop: true`: habilita interoperabilidad entre `import` y CommonJS.
- `forceConsistentCasingInFileNames: true`: exige mayúsculas consistentes en rutas.
- `strict: true`: habilita chequeo estricto de tipos.
- `skipLibCheck: true`: omite verificación de tipos en librerías externas.

7) src/views/Cl_vAspirante.ts

Implementa la vista del formulario evaluador.

Importa `I_vAspirante`.

En el constructor:
- captura referencias a los inputs del DOM por id.
- captura botones `Enviar` y `Cancelar`.
- captura el contenedor `aspirante`.
- llama `aplicarValidaciones()` para forzar min/max en inputs.

Getters:
- `nombre`, `cedula`, `notaExamenEscrito`, `notaExamenPractico`, `notaExamenAptitudes`.

Métodos de puntaje:
- `puntajeCO5()`: lee inputs 1 a 4; no acepta valores negativos; retorna valores positivos.
- `puntajeCO51()`: lee inputs 5 a 11.
- `puntajeCO52()`: lee inputs 12 a 20.
- `puntajeCO53()`: lee inputs 21 a 28.

Eventos:
- `onEnviar(callback)`: asigna callback a `btnEnviar.onclick`.
- `onCancelar(callback)`: asigna callback a `btnCancelar.onclick`.

Visibilidad:
- `mostrar()`: muestra la vista y limpia valores.
- `ocultar()`: oculta la vista.

Confirmación:
- `mostrarConfirmacion()`: reemplaza el contenido de la sección con un mensaje de éxito y botón para volver a `indexEvaluador.html`.

Validaciones:
- `aplicarValidaciones()`: agrega evento `input` a los 28 campos de preguntas.
- Comprueba que el valor sea numérico y respete `min` y `max`.
- Si el valor excede el máximo, lo corrige y alerta.
- Si el valor es menor que el mínimo, lo corrige y alerta.

10) src/controllers/Cl_cAspirante.ts

Controlador del evaluador.
- Importa `I_vAspirante`, `Cl_mAspirante` y `sAspirante`.
- Guarda `vista` en una propiedad privada.
- En el constructor, liga `vista.onEnviar()` al método `btEnviarOnClick()`.

`btEnviarOnClick()`:
- Crea `aspirante` nuevo con los datos básicos del formulario.
- Llama a `vista.puntajeCO5()`, `puntajeCO51()`, `puntajeCO52()` y `puntajeCO53()`.
- Para cada puntaje devuelto, invoca `aspirante.agregarPuntos(...)` con la sección correspondiente.
- Llama a `sAspirante.agregar(aspirante)` y espera el resultado.
- Muestra un `alert` con `resultado.mensaje`.
- Si `resultado.ok` es verdadero, llama a `vista.mostrarConfirmacion()`.

Este controlador es un coordinador: no realiza cálculos de negocio, sólo usa la vista y el modelo.

13) src/models/Cl_mAspirante.ts

Modelo del aspirante con lógica de negocio.

Atributos:
- `tabla`: cadena fija "aspirante".
- `_nombre`, `_cedula`, `_notaExamenEscrito`, `_notaExamenPractico`, `_notaExamenAptitudes`.
- Arrays `ptsFormatoCO5`, `ptsFormatoCO51`, `ptsFormatoCO52`, `ptsFormatoCO53`.
- Arrays `juradoACO10`, `juradoBCO10`, `juradoCCO10` para futura tabla CO10.

Constructor:
- recibe un objeto con nombre, cédula y 3 notas.
- inicializa los setters correspondientes.

Getters y setters:
- validan el tipo de nota con `+n` para convertir a número.

`agregarPuntos(seccion, puntos)`:
- según la sección, agrega puntos al array correspondiente.
- si la sección no coincide, muestra error en consola.

Métodos de suma:
- `sumaPtsFormatoCO5()`: suma elementos de `ptsFormatoCO5`, corta a 35 si excede.
- `sumaPtsFormatoCO51()`: suma elementos de `ptsFormatoCO51`, corta a 30.
- `sumaPtsFormatoCO52()`: suma y limita con `Math.min(suma, 15)`.
- `sumaPtsFormatoCO53()`: suma y limita con `Math.min(suma, 20)`.

`totalObtenido()`:
- suma los resultados de los cuatro bloques.

`calificacionFinal()`:
- divide `totalObtenido()` entre 5 para obtener nota sobre 20.

`cargarSumasGuardadas(sumas)`:
- si los arrays de puntos están vacíos, los rellena con las sumas leídas de MockAPI.
- usa `Number(...) || 0` para asegurar valor numérico.

Sección CO7/CO8/CO9/CO10:
- `calificacion10Porciento()`: totalObtenido * 10%.
- `calificacionCO8()`: promedio de examen escrito y práctico.
- `calificacion60PorcientoCO8()`: promedio CO8 * 60%.
- `calificacion30PorcientoAptitudes()`: nota de aptitudes * 30%.
- `agregarPtsJurado(jurado, puntosJurado)`: almacena puntajes de jurados en arrays; aún no usado.
- `notaDefinitiva()`: suma CO7 + CO8 + CO9.

`toJSON()`:
- prepara un objeto plano para guardar en MockAPI.
- agrega `tabla`, `nombre`, `cedula`, suma de formatos y calificaciones.

17) src/services/Cl_sMockApi.ts

Capa base para comunicación con MockAPI.

Atributos:
- `apiUrl`: URL de la API. Si está vacía, retorna error.

`fetchMockApi({...})`:
- configura `RequestInit` con `method`, `headers` y `body` si existe.
- realiza `fetch(uri, options)`.
- maneja `404` devolviendo `ok: true` y `data: []`.
- si la respuesta no es ok, devuelve `ok: false`.
- intenta parsear JSON; si falla, devuelve `data: null`.
- en errores de red devuelve `ok: false` y `message`.

`getTabla({ tabla })`:
- arma `uri = this.apiUrl`.
- pide GET a la API.
- si responde 404 devuelve tabla vacía.
- si no es ok devuelve error.
- retorna `data` o array vacío.

`existeId({ tabla, tablaId, tablaIdName })`:
- construye una URL con query `?tabla=...&campo=...`.
- si la respuesta es 404, asume que no existe el registro.
- si hay error real, devuelve ok false.
- si hay datos y la respuesta es 200, devuelve `true` si hay al menos un registro.

`agregar(registro)`:
- POST a la API con el cuerpo JSON.
- si hay error devuelve mensaje.
- si guarda correctamente devuelve mensaje con ID.

`obtenerIdMockApi(...)`:
- busca el recurso en MockAPI con `tablaIdName` y valor.
- si hay resultados, devuelve el `id` interno de MockAPI.
- útil para PUT y DELETE.

`modificar(tablaId, registro, tablaIdName)`:
- obtiene `recursoId` con `obtenerIdMockApi`.
- si no lo encuentra, devuelve error.
- hace PUT a `apiUrl/recursoId`.
- devuelve mensaje.

`eliminar(tablaId, tabla, tablaIdName)`:
- obtiene `recursoId`.
- hace DELETE a `apiUrl/recursoId`.
- devuelve mensaje.

25) Notas importantes sobre el proyecto

- La interfaz `indexEvaluador.html` tiene un error tipográfico en `hiddenn` que podría impedir ocultar la sección correctamente.
- El flujo principal de registro guarda únicamente las sumas por bloque, no cada pregunta individual.
- La lógica de cálculo del modelo está centralizada en `Cl_mAspirante`.
- Manualmente se construyen los arrays de puntajes desde la vista y luego se agregan al modelo.
- El controlador `Cl_cEntregas` recarga datos desde MockAPI y los transforma en objetos modelo para renderizar la tabla.
