16-06-2026 RESUMENDEFUNCIONAMIENTO

Este archivo resume lo esencial para entender el funcionamiento del proyecto "L26.p.c3.01.Concurso-Laboral".

1) PROPÓSITO GENERAL

El proyecto es una aplicación web para un concurso de aspirantes. Permite:
- Al evaluador: ingresar datos de aspirantes, puntuar 28 preguntas y guardar resultados en MockAPI.
- Al administrador: consultar las entregas guardadas y verlas en una tabla.

2) ESTRUCTURA

Capas principales:
- HTML: define la interfaz de evaluador y administrador.
- Views: leen/escriben el DOM y exponen eventos.
- Controllers: coordinan eventos, modelos y servicios.
- Models: contienen la lógica de negocio y cálculos.
- Services: comunican con MockAPI.

3) ARCHIVOS CLAVE

- `indexEvaluador.html`: interfaz del evaluador.
- `indexAdministrador.html`: interfaz del administrador.
- `src/views/Cl_vAspirante.ts`: vista del formulario evaluador.
- `src/views/Cl_vConcurso.ts`: vista del administrador.
- `src/views/Cl_vEntregas.ts`: vista de la tabla de entregas.
- `src/controllers/Cl_cAspirante.ts`: controlador de ingreso y guardado de aspirantes.
- `src/controllers/Cl_cConcurso.ts`: controlador de navegación hacia entregas.
- `src/controllers/Cl_cEntregas.ts`: controlador de carga y renderizado de entregas.
- `src/models/Cl_mAspirante.ts`: modelo de aspirante con cálculos de puntaje.
- `src/models/Cl_mConcurso.ts`: modelo contenedor de aspirantes.
- `src/services/Cl_sMockApi.ts`: capa base de comunicación HTTP con MockAPI.
- `src/services/Cl_sProyecto.ts`: define la URL del proyecto MockAPI.
- `src/services/Cl_sAspirante.ts`: servicio para validar y agregar aspirantes.
- `src/services/Cl_sEntregas.ts`: servicio para obtener entregas.
- `indexAdministrador.html`: ahora también incluye un botón `Reporte` dentro de la sección de entregas y una sección `reporte` oculta con métricas de resumen.

4) FUNCIONAMIENTO ESENCIAL

Flujo del evaluador:
- El usuario ingresa nombre, cédula, tres notas y 28 puntajes de preguntas.
- `Cl_cAspirante.btEnviarOnClick()` crea instancia `Cl_mAspirante` con los datos básicos.
- Se leen los puntajes agrupados en cuatro métodos de la vista:
  - `puntajeCO5()`: preguntas 1-4.
  - `puntajeCO51()`: preguntas 5-11.
  - `puntajeCO52()`: preguntas 12-20.
  - `puntajeCO53()`: preguntas 21-28.
- Cada puntaje se agrega al modelo mediante `aspirante.agregarPuntos(seccion, puntos)`.
- Se llama `sAspirante.agregar(aspirante)` para validar y guardar en MockAPI.
- Si la operación es exitosa, muestra confirmación en la vista.

Flujo del administrador:
- `Cl_cConcurso` escucha el botón "Ver Entregas".
- Al hacer clic se crea `Cl_vEntregas`, `Cl_mConcurso` y `Cl_cEntregas`.
- `Cl_cEntregas.btRecargarOnClick()` obtiene datos con `Cl_sEntregas.getEntregas()`.
- `modelo.setAspirantes(resultado.tabla)` transforma cada registro en `Cl_mAspirante`.
- La vista muestra filas con los datos procesados.
- El botón `Reporte` dentro de la sección de entregas abre una vista resumida sin mostrar la tabla, con porcentaje de calificación y aspirantes con más de 25 puntos en CO5.

5) ESPECIFICACIONES DE MÉTODOS PRINCIPALES

`Cl_vAspirante`:
- `get nombre()`: devuelve el valor del input nombre.
- `get cedula()`: devuelve el valor del input cédula.
- `get notaExamenEscrito()`: devuelve la nota del examen escrito.
- `get notaExamenPractico()`: devuelve la nota del examen práctico.
- `get notaExamenAptitudes()`: devuelve la nota del examen de aptitudes.
- `puntajeCO5()`: lee preguntas 1-4, valida no negativos y retorna valores positivos.
- `puntajeCO51()`: lee preguntas 5-11, valida no negativos.
- `puntajeCO52()`: lee preguntas 12-20, valida no negativos.
- `puntajeCO53()`: lee preguntas 21-28, valida no negativos.
- `onEnviar(callback)`: asigna callback al evento clic del botón Enviar.
- `onCancelar(callback)`: asigna callback al botón Cancelar.
- `mostrar()`: muestra la sección del formulario y limpia los campos.
- `ocultar()`: oculta la sección.
- `mostrarConfirmacion()`: reemplaza la vista con un mensaje de registro exitoso.
- `aplicarValidaciones()`: agrega validación de min/max a los inputs de preguntas.

`Cl_vConcurso`:
- `onVerEntregas(callback)`: asigna callback al botón Ver Entregas.
- `deshabilitarBotones()`: desactiva el botón.
- `habilitarBotones()`: activa el botón.

`Cl_vEntregas`:
- `mostrarAspirantes(aspirantes)`: genera filas HTML con datos de cada aspirante.
- `onRecargar(callback)`: asigna callback al botón Recargar.
- `onVolver(callback)`: asigna callback al botón Volver.
- `mostrar()`: muestra la sección de entregas.
- `ocultar()`: oculta la sección.

`Cl_cAspirante`:
- `btEnviarOnClick()`: método principal que crea el modelo, agrega puntajes y llama al servicio para guardar.

`Cl_cConcurso`:
- `onVerEntregas()`: crea la vista y controlador de entregas y deshabilita el botón principal.

`Cl_cEntregas`:
- `onVolver()`: oculta la vista de entregas y ejecuta callback para regresar.
- `btRecargarOnClick()`: obtiene entregas desde el servicio, carga el modelo y actualiza la vista.

`Cl_mAspirante`:
- `agregarPuntos(seccion, puntos)`: añade el puntaje al array de la sección correcta.
- `sumaPtsFormatoCO5()`: suma puntos de CO5 y aplica tope 35.
- `sumaPtsFormatoCO51()`: suma puntos de CO5.1 y aplica tope 30.
- `sumaPtsFormatoCO52()`: suma puntos de CO5.2 y limita a 15.
- `sumaPtsFormatoCO53()`: suma puntos de CO5.3 y limita a 20.
- `totalObtenido()`: suma las cuatro secciones.
- `calificacionFinal()`: divide el total obtenido entre 5 para escala de 20.
- `cargarSumasGuardadas(sumas)`: restaura totales desde un objeto de datos externos.
- `calificacion10Porciento()`: calcula 10% de `totalObtenido()`.
- `calificacionCO8()`: calcula el promedio de notas escrito y práctico.
- `calificacion60PorcientoCO8()`: aplica 60% al promedio CO8.
- `calificacion30PorcientoAptitudes()`: aplica 30% a la nota de aptitudes.
- `notaDefinitiva()`: suma CO7 + CO8 + CO9.
- `toJSON()`: prepara los datos para enviar a la API, incluyendo totales y calificación final.

`Cl_mConcurso`:
- `agregarAspirante(aspirante)`: inserta un objeto en `aspirantes`.
- `setAspirantes(aspirante)`: convierte un array de objetos obtenidos en instancias `Cl_mAspirante`.
- `getAspirantes()`: devuelve el array.

`Cl_sMockApi`:
- `fetchMockApi({...})`: realiza solicitudes fetch y maneja respuestas HTTP.
- `getTabla({ tabla })`: obtiene la lista completa de registros.
- `existeId({ tabla, tablaId, tablaIdName })`: verifica existencia de un registro por campo.
- `agregar(registro)`: crea un registro nuevo con POST.
- `obtenerIdMockApi({...})`: busca el `id` interno de MockAPI.
- `modificar(tablaId, registro, tablaIdName)`: actualiza un registro existente.
- `eliminar(tablaId, tabla, tablaIdName)`: elimina un registro.

`Cl_sProyecto`:
- define `apiUrl` del proyecto MockAPI.

`Cl_sAspirante`:
- `existe(aspiranteId)`: revisa si ya hay un aspirante con la cédula dada.
- `agregar(nuevoAspirante)`: valida campos, unicidad y guarda el aspirante.

`Cl_sEntregas`:
- `getEntregas()`: devuelve todos los registros de aspirantes.

6) PUNTOS IMPORTANTES

- El sistema guarda en la API solo las sumas de cada bloque, no cada pregunta individual.
- La calificación final mostrada en la tabla es `totalObtenido() / 5`.
- Se usan getters en la vista para leer valores de los inputs al vuelo.
- Los servicios usan `MockAPI` con URL fija en `Cl_sProyecto`.
- El archivo `resources/08-06-2026 RESUMENDEFUNCIONAMIENTO.txt` acompaña el informe instructivo ya creado.
